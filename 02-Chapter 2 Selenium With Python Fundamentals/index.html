üåü 2.1 What is Selenium?

Selenium is an open-source automation tool used for testing web applications across different browsers and platforms. It mimics real user actions like clicking buttons, typing in forms, navigating links, etc.

‚úÖ Why Selenium?

  • Supports multiple programming languages: Python, Java, C#, Ruby
  • Works on multiple browsers: Chrome, Firefox, Edge, Safari
  • Can be integrated with test frameworks like Pytest, Unittest, etc.
  • Supports parallel execution and CI/CD tools (e.g., Jenkins)

üß† 2.2 Selenium Architecture Overview

Selenium has four main components:

  • Selenium IDE ‚Äì A browser plugin for recording scripts.
  • Selenium RC ‚Äì Legacy tool (Deprecated).
  • Selenium WebDriver ‚Äì Modern, robust automation framework.
  • Selenium Grid ‚Äì For distributed test execution.

‚úÖ 1. Selenium IDE

üõ† What is it?
Selenium IDE is a browser extension (for Chrome and Firefox) that allows recording, editing, and debugging of functional tests.

  • Click ‚ÄúRecord‚Äù ‚Üí It captures user actions like clicking, typing, etc.
  • It generates test cases in a tabular or script format.
  • Tests can be replayed directly.

‚ùå 2. Selenium RC (Remote Control)

üõ† What is it?
Selenium RC was the original solution for automating browsers before WebDriver.

‚úÖ Why deprecated: Required server setup, slower, replaced by WebDriver.

üöÄ 3. Selenium WebDriver ‚Äì Modern Standard

Allows direct control of browsers through language-specific bindings.

üß™ 2.4 First Selenium Test Script in Python

‚úÖ Test Case: Open Google and search for "Selenium"

from selenium import webdriver
from selenium.webdriver.common.by import By
import time

driver = webdriver.Chrome()
driver.get("https://www.google.com")

search_box = driver.find_element(By.NAME, "q")
search_box.send_keys("Selenium")
search_box.submit()

time.sleep(5)
driver.quit()

üîç 2.5 Locators in Selenium

Locators help Selenium find elements on a webpage.

Common Locator Strategies:

  • ID ‚Äì find_element(By.ID, "username")
  • Name ‚Äì find_element(By.NAME, "email")
  • Class Name ‚Äì find_element(By.CLASS_NAME, "btn")
  • Tag Name ‚Äì find_element(By.TAG_NAME, "input")
  • Link Text ‚Äì find_element(By.LINK_TEXT, "Click Here")
  • Partial Link Text ‚Äì find_element(By.PARTIAL_LINK_TEXT, "Click")
  • XPath ‚Äì find_element(By.XPATH, "//input[@id='search']")
  • CSS Selector ‚Äì find_element(By.CSS_SELECTOR, "input[name='q']")

‚úÖ Examples of Each Locator

# ID
driver.find_element(By.ID, "username").send_keys("admin")

# Name
driver.find_element(By.NAME, "password").send_keys("admin123")

# Class Name
driver.find_element(By.CLASS_NAME, "login-button").click()

# Tag Name
all_links = driver.find_elements(By.TAG_NAME, "a")
for link in all_links:
    print(link.text)

# Link Text
driver.find_element(By.LINK_TEXT, "Forgot Password?").click()

# Partial Link Text
driver.find_element(By.PARTIAL_LINK_TEXT, "Forgot").click()

ü§î Why use XPath?

  • When ID or class names are missing or dynamic
  • When elements are deeply nested
  • When you want to select elements based on text, relationships, or conditions

üß† Types of XPath

1Ô∏è‚É£ Absolute XPath ‚ùå

It starts from the root of the HTML document and goes through each node until the desired element.

/html/body/div[1]/form/input[2] ‚Äì Brittle, not recommended

2Ô∏è‚É£ Relative XPath ‚úÖ

Uses double forward slashes to search anywhere in the document.

//tagname[@attribute='value']

Example:

driver.find_element(By.XPATH, "//input[@id='username']")

üîç XPath Syntax Variants

//input[@name='email']
//input[contains(@name, 'user')]
//input[starts-with(@id, 'user')]
//button[text()='Submit']
//a[contains(text(), 'Login')]

üîó XPath Axes

//input[@id='email']/parent::div
//label[text()='Username']/following-sibling::input
//input[@id='password']/preceding-sibling::label
//span[text()='Login']/ancestor::form

üõ† Real World Example

<div class="form">
  <label for="email">Email:</label>
  <input type="text" name="email" id="emailInput" />
</div>

Example 1:

driver.find_element(By.XPATH, "//input[@id='emailInput']")

Example 2:

driver.find_element(By.XPATH, "//label[text()='Email:']/following-sibling::input")

üö® Common Mistakes

  • ‚ùå Using absolute XPath
  • ‚ùå Typos in tag/attribute
  • ‚ùå Not escaping quotes

‚úÖ Best Practices

  • Use relative XPath
  • Use contains()/starts-with()
  • Avoid deep absolute paths
